# Анализ стоимости использования Apify акторов для GMGN данных

## Акторы для использования

### 1. GMGN Token Stat Scraper
- **URL**: https://apify.com/muhammetakkurtt/gmgn-token-stat-scraper
- **Модель ценообразования**: Pay per event
- **Поддержка BSC**: ✅ Да
- **Возвращаемые данные**: Comprehensive token information (price, liquidity, holders, trading metrics, developer info)

### 2. GMGN Trending Data
- **URL**: https://apify.com/fusioninsight/gmgn-trending-data
- **Модель ценообразования**: $0.005 / single trending record
- **Поддержка BSC**: ✅ Да
- **Возвращаемые данные**: Trending tokens with risk metrics

## Анализ использования бота

### Текущая логика
- **Интервал сканирования**: 30 секунд
- **Циклов в час**: 120 циклов
- **Токенов проверяется за цикл**: ~50-200 (из разных источников)
- **Токенов проходят базовые фильтры**: ~5-20 за цикл
- **Токенов проходят все фильтры и требуют GMGN проверки**: ~1-5 за цикл (только те, что готовы к алерту)

### Когда вызывать Apify
Apify нужно вызывать **только для токенов, которые прошли все фильтры** и готовы к отправке алерта. Это происходит в двух местах:

1. **Established токены** (volume spike >= $2M) - ~1-2 токена в час
2. **Новые токены** (volume >= $300K, age < 2h) - ~2-10 токенов в час

**Итого**: ~3-12 токенов в час требуют проверки GMGN

## Полезные данные из GMGN Token Stat Scraper

Из документации видно, что актор возвращает:
- `dev.top_10_holder_rate` - процент у топ-10 holders (уже есть в DexTools)
- `dev.creator_address` - адрес создателя (можно использовать для проверки banned addresses)
- `price.*` - детальная информация о цене и объемах
- `pool.*` - информация о пуле ликвидности
- `priority_fee`, `tip_fee` - комиссии (может быть связано с bundler)

**⚠️ ВАЖНО**: В документации **НЕ указано**, что актор возвращает bundler % и total fees для BSC токенов. Эти данные могут быть доступны только через парсинг HTML страницы GMGN или через другой источник.

## Рекомендации по интеграции

### Вариант 1: Использовать Apify для получения comprehensive данных
- Использовать **GMGN Token Stat Scraper** для получения всех доступных данных
- Проверить, есть ли bundler/total fees в реальных ответах API
- Если нет - использовать как дополнительный источник данных для улучшения фильтрации

### Вариант 2: Комбинированный подход
- Использовать Apify для получения данных о токене
- Парсить HTML только для bundler % и total fees (если они критичны)
- Кешировать результаты Apify на 1-2 часа

### Вариант 3: Использовать только Apify (если данные там есть)
- Полностью заменить парсинг HTML на Apify API
- Более надежно и стабильно
- Но нужно проверить наличие bundler/total fees в ответах
