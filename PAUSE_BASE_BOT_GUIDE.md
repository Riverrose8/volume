# ⏸️ Приостановка Base бота на 6 часов

## 🚀 Быстрая приостановка

### Вариант 1: Простой скрипт
```bash
python3 quick_pause_base.py
```
Введите IP адрес VPS и бот будет остановлен на 6 часов.

### Вариант 2: Полный скрипт с автоматическим возобновлением
```bash
python3 pause_base_bot.py
```
Выберите опцию 1 для приостановки с автоматическим возобновлением.

### Вариант 3: Принудительная остановка (если обычная не работает)
```bash
python3 force_stop_base_bot.py
```
Используйте, если обычная остановка не работает.

### Вариант 4: Диагностика проблем
```bash
python3 diagnose_base_bot.py
```
Для диагностики и исправления проблем с сервисом.

## 🔧 Ручная приостановка

### Остановка бота:
```bash
ssh root@YOUR_VPS_IP "systemctl stop base-bot"
```

### Проверка статуса:
```bash
ssh root@YOUR_VPS_IP "systemctl is-active base-bot"
```
Должно показать: `inactive`

## ▶️ Возобновление

### Вариант 1: Через скрипт
```bash
python3 pause_base_bot.py
```
Выберите опцию 2 для немедленного возобновления.

### Вариант 2: Ручное возобновление
```bash
ssh root@YOUR_VPS_IP "systemctl start base-bot"
```

### Проверка статуса:
```bash
ssh root@YOUR_VPS_IP "systemctl is-active base-bot"
```
Должно показать: `active`

## 📊 Что происходит при приостановке

1. ✅ **Base бот останавливается** - перестает сканировать токены
2. ✅ **USDC пары не отслеживаются** - CGN и другие токены не будут обнаружены
3. ✅ **Алерты не отправляются** - канал будет тихим
4. ✅ **Ресурсы освобождаются** - CPU и память не используются

## ⏰ Автоматическое возобновление

При использовании полного скрипта:
- ✅ **Автоматическое возобновление** через 6 часов
- ✅ **Логи возобновления** в `/root/resume_bot.log`
- ✅ **Проверка каждую минуту** до времени возобновления

## 🔍 Мониторинг

### Проверка логов возобновления:
```bash
ssh root@YOUR_VPS_IP "tail -f /root/resume_bot.log"
```

### Проверка статуса сервиса:
```bash
ssh root@YOUR_VPS_IP "systemctl status base-bot"
```

### Проверка последних логов бота:
```bash
ssh root@YOUR_VPS_IP "journalctl -u base-bot -n 20"
```

## ⚠️ Важные моменты

1. **Пропущенные токены**: Во время приостановки могут быть пропущены качественные токены
2. **USDC пары**: CGN и другие USDC пары не будут обнаружены
3. **Ручное возобновление**: Можно возобновить в любое время командой `systemctl start base-bot`
4. **Автоматическое возобновление**: Работает только при использовании полного скрипта

## 🚨 Устранение неполадок

### Если обычная остановка не работает:

1. **Диагностика проблем:**
   ```bash
   python3 diagnose_base_bot.py
   ```

2. **Принудительная остановка:**
   ```bash
   python3 force_stop_base_bot.py
   ```

3. **Ручная остановка через SSH:**
   ```bash
   # Остановка сервиса
   ssh root@YOUR_VPS_IP "systemctl stop base-bot"
   
   # Завершение процессов
   ssh root@YOUR_VPS_IP "pkill -f main_base.py"
   
   # Отключение сервиса
   ssh root@YOUR_VPS_IP "systemctl disable base-bot"
   ```

### Если сервис не отвечает:

1. **Перезагрузка systemd:**
   ```bash
   ssh root@YOUR_VPS_IP "systemctl daemon-reload"
   ```

2. **Проверка конфигурации:**
   ```bash
   ssh root@YOUR_VPS_IP "systemctl cat base-bot"
   ```

3. **Проверка логов:**
   ```bash
   ssh root@YOUR_VPS_IP "journalctl -u base-bot -n 50"
   ```

Base бот снова будет:

- ✅ **Обнаруживать токены** как CGN
- ✅ **Отправлять алерты** в канал
- ✅ **Применять все фильтры** качества
