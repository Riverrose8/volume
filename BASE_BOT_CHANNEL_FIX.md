# 🔧 Base Bot Channel Configuration Fix

## 🎯 Проблема решена!

Base бот теперь настроен на **тестовый канал** вместо основного PancakeSwap канала.

### ✅ Что было исправлено:

1. **Остановили Base бота** - чтобы избежать дальнейших алертов в неправильный канал
2. **Изменили TELEGRAM_CHAT_ID** с `-1002960789020` (PancakeSwap) на `-1003119685017` (тестовый)
3. **Перезапустили Base бота** с новыми настройками

### 📊 Текущие настройки Base бота:

- **TELEGRAM_CHAT_ID**: `-1003119685017` (тестовый канал)
- **TELEGRAM_TEST_CHAT_ID**: `-1003119685017` (тот же тестовый канал)
- **TEST_MODE**: `False`

### 🎯 Рекомендации:

#### Вариант 1: Использовать тестовый канал
- Base алерты будут приходить в тестовый канал `-1003119685017`
- Это безопасно и не мешает основному PancakeSwap боту

#### Вариант 2: Создать отдельный канал для Base
Если хочешь создать отдельный канал для Base алертов:

1. **Создай новый Telegram канал** для Base алертов
2. **Получи ID канала** (можно через @userinfobot)
3. **Обнови настройки**:
   ```bash
   ssh ubuntu@84.201.161.90 "cd /home/ubuntu/pancake-bot && sed -i 's/TELEGRAM_CHAT_ID=-1003119685017/TELEGRAM_CHAT_ID=НОВЫЙ_ID_КАНАЛА/' .env_base"
   ```
4. **Перезапусти бота**:
   ```bash
   ssh ubuntu@84.201.161.90 "sudo systemctl restart base-bot"
   ```

### 🔍 Проверка работы:

```bash
# Проверить статус
ssh ubuntu@84.201.161.90 "sudo systemctl status base-bot"

# Посмотреть логи
ssh ubuntu@84.201.161.90 "sudo journalctl -u base-bot -f"

# Проверить настройки
ssh ubuntu@84.201.161.90 "grep TELEGRAM_CHAT_ID /home/ubuntu/pancake-bot/.env_base"
```

### ✅ Результат:

- **PancakeSwap бот** → основной канал `-1002960789020`
- **Base бот** → тестовый канал `-1003119685017`
- **Никаких конфликтов** между ботами!
